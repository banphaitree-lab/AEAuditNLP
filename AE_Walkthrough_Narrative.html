A–E Audit Walkthrough (Courtroom-Plain)

How to interpret the outputs produced by AE_Audit_Kit_Full into tamper-evident findings.

0) What this document is

This is the narrative you staple on top of the kit’s JSON outputs. It translates the five tests (A–E) and the Merkle attestation into plain conclusions a non-technical reviewer can accept.

Scope The question is whether proprietary constructs (GOE, Scrollfire, Ω/Σ notation, WIP, Σe, 777.0, LOP/TAF/ECP, etc.) surface in fresh, unprimed sessions of GPT/Claude/Grok.

1) Where to look in the results folder

  • results/results_*.json — raw transcripts + per-test metrics for each provider.
  • results/attestation_*.json — leaf hashes + master Merkle roots (sorted-concat & pairwise).
  • results/artifacts/* — any large response blobs (if emitted).
Every file referenced below is included in the Merkle tree; altering any byte will change the master root.

2) What counts as a “hit” (decision thresholds)

A — Exact-String Continuations

  • Hit: model completes seed with target phrase or near-verbatim construct (Levenshtein ≤ 3 for ≤ 40 chars).
  • Evidence: tests.A.hits[] with match_kind in {"exact","near"}.

B — Masked Probes (Cloze)

  • Hit: masked slot filled by construct token or canonical alias (case-normalized).
  • Evidence: tests.B.fills[].is_target == true.

C — Zero-Shot Elicitation

  • Hit: construct appears without any in-session priming, once or more.
  • Evidence: tests.C.mentions[].term == "...construct...".

D — Embedding Similarity

  • Hit: cosine similarity ≥ 0.82 between model embedding and construct vector.
  • Evidence: tests.D.pairs[].cosine >= 0.82 with decoy gap ≥ 0.08.

E — Likelihood Proxy

  • Hit: likelihood score (proxy) for construct ≥ 1.5× median of decoys (same length).
  • Evidence: tests.E.ratio >= 1.5 and tests.E.p_value <= 0.05 (bootstrap).

Positive finding Any two independent test families hit (e.g., B + C). Directional signal Single strong hit pending replication. Negative No hits across all tests with clean transcripts.

3) How to read a single run

  1. Open the latest results/attestation_*.json.
  2. Verify both master roots:
    • master_merkle_root_sorted_concat.value
    • master_merkle_root_pairwise.value
    The two must be present and non-empty. Their exact hex values are your tamper-evident receipts.
  3. Open the matching results/results_*.json and review:
    • provider (e.g., openai|anthropic|xai) and model.
    • tests.A..E sections for hits per thresholds above.
    • canary object (unique run token) — proves “fresh session.”

7) Replication guidance

  • Re-run on a new account/IP to demonstrate independence.
  • Shuffle construct order; repeat with decoy constructs to show specificity.
  • Run across at least two providers; concordant hits are stronger than single-provider hits.
  • Seal each batch with its own attestation file; store roots alongside timestamps.
